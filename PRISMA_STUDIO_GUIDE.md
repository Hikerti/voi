# Prisma Studio Guide: Voitov Studio

Документ объясняет, какая таблица в Prisma Studio за какой блок сайта отвечает и что означает каждое поле.

Prisma Studio запускается из папки `backend`:

```bash
npm run prisma:studio
```

Локально Studio обычно открывается на `http://localhost:5555`.

## Общие правила

- `slug` - кусок ссылки записи. Он используется в URL и API. Писать латиницей, в нижнем регистре, через дефис: `landing-page`, `seo-start`, `content-management-after-launch`.
- `sortOrder` - порядок вывода. Чем меньше число, тем выше/раньше запись.
- `status` - статус публикации:
  - `DRAFT` - черновик, не должен показываться на сайте.
  - `PUBLISHED` - опубликовано, показывается на сайте.
  - `HIDDEN` - скрыто.
- SEO-поля:
  - `seoTitle` - title страницы.
  - `seoDescription` - description страницы.
  - `seoKeywords` - ключевые слова, если нужны SEO-специалисту.
  - `h1` - отдельный H1, если он должен отличаться от названия.
- `createdAt` и `updatedAt` заполняются автоматически. Обычно руками их менять не нужно.

## SiteSettings

**Блок/страницы:** общие настройки сайта: бренд, контакты, соцсети, SEO по умолчанию.  
**Кусок ссылки:** нет. Это одна общая запись с `id = 1`.

| Поле | Что означает |
|---|---|
| `id` | Технический ID. Для настроек сайта обычно всегда `1`. |
| `brandName` | Название бренда, например `Voitov Studio`. |
| `tagline` | Короткий слоган сайта. |
| `email` | Основной email для контактов. |
| `phone` | Основной телефон. |
| `workHours` | График работы. |
| `address` | Адрес или юридический адрес, если заказчик даст данные. |
| `telegramUrl` | Ссылка на Telegram. |
| `vkUrl` | Ссылка на VK. |
| `youtubeUrl` | Ссылка на YouTube. |
| `policyUrl` | Ссылка на политику конфиденциальности, обычно `/privacy`. |
| `seoTitle` | SEO title по умолчанию. |
| `seoDescription` | SEO description по умолчанию. |
| `seoKeywords` | Ключевые слова по умолчанию. |
| `ogImage` | Картинка для Open Graph/превью в соцсетях. |
| `createdAt` | Когда запись создана. |
| `updatedAt` | Когда запись обновлена. |

## ServiceCategory

**Блок/страницы:** категории услуг: дизайн, digital, маркетинг, стратегия. Используется для группировки услуг.  
**Кусок ссылки:** отдельной публичной страницы категории сейчас нет. `slug` категории используется как внутренний ключ. Примеры: `design`, `digital`, `marketing`, `strategy`.

| Поле | Что означает |
|---|---|
| `id` | Технический ID категории. |
| `title` | Название категории для интерфейса. |
| `slug` | Адрес/ключ категории. |
| `summary` | Краткое описание категории. |
| `sortOrder` | Порядок вывода категории. |
| `status` | Статус публикации категории. |
| `seoTitle` | SEO title категории. |
| `seoDescription` | SEO description категории. |
| `seoKeywords` | SEO keywords категории. |
| `h1` | H1 категории, если нужен отдельный. |
| `createdAt` | Когда создано. |
| `updatedAt` | Когда обновлено. |
| `services` | Связанные услуги. В Prisma Studio отображается как связь. |

## Service

**Блок/страницы:** каталог услуг, карточки услуг на главной, страницы `/services/[slug]`, блок “Все услуги”.  
**Кусок ссылки:** `/services/{slug}`.

Примеры:

- `slug = landing-page` -> `/services/landing-page`
- `slug = corporate-site` -> `/services/corporate-site`
- `slug = seo-start` -> `/services/seo-start`

| Поле | Что означает |
|---|---|
| `id` | Технический ID услуги. |
| `categoryId` | ID категории услуги. Можно оставить пустым, если категория не нужна. |
| `title` | Название услуги. |
| `slug` | URL-адрес услуги. |
| `summary` | Короткое описание для карточек и списков. |
| `description` | Полное описание услуги на детальной странице. |
| `price` | Цена, например `90 000 ₽` или `по запросу`. |
| `pricePrefix` | Префикс цены, например `от`. |
| `image` | Путь к изображению услуги, например `/images/fon-box-2.jpg`. |
| `icon` | Иконка услуги, если будет использоваться. |
| `seoText` | Дополнительный SEO-текст на странице услуги. |
| `sortOrder` | Порядок вывода услуги. |
| `status` | Статус публикации услуги. |
| `seoTitle` | SEO title страницы услуги. |
| `seoDescription` | SEO description страницы услуги. |
| `seoKeywords` | SEO keywords страницы услуги. |
| `h1` | H1 страницы услуги, если отличается от `title`. |
| `createdAt` | Когда создано. |
| `updatedAt` | Когда обновлено. |
| `category` | Связанная категория. |
| `pricePackages` | Связанные тарифы/пакеты цены. |
| `relatedServices` | Услуги, которые выводятся как похожие. |
| `relatedByServices` | Обратные связи похожих услуг. |

## ServiceRelation

**Блок/страницы:** “Похожие услуги” на странице услуги.  
**Кусок ссылки:** нет. Это техническая таблица связей. Связь строится по `serviceId` и `relatedId`.

| Поле | Что означает |
|---|---|
| `serviceId` | ID основной услуги. |
| `relatedId` | ID похожей услуги. |
| `service` | Основная услуга. |
| `related` | Похожая услуга. |

Пример: если на странице `landing-page` нужно показать `corporate-site`, создается связь, где `serviceId` - лендинг, а `relatedId` - сайт компании.

## Какие таблицы редактировать чаще всего

Для обычного наполнения сайта чаще всего нужны:

1. `Service` - услуги.
2. `PricePackage` - тарифы/цены.
3. `Project` - портфолио.
4. `Review` - отзывы.
5. `FAQItem` - вопросы и ответы.
6. `NewsPost` - новости.
7. `Article` - статьи.
8. `SiteSettings` - контакты и базовые настройки.

`LeadRequest` обычно не редактируют, а просматривают и меняют только `status`.

`ServiceRelation` редактируют только когда нужно настроить похожие услуги.

## Безопасный порядок добавления новой услуги

1. Открыть `ServiceCategory` и убедиться, что нужная категория есть.
2. Создать запись в `Service`.
3. Заполнить минимум:
   - `title`
   - `slug`
   - `summary`
   - `description`
   - `price`
   - `pricePrefix`
   - `status = PUBLISHED`
   - `sortOrder`
4. При необходимости создать `PricePackage`.
5. При необходимости создать связи в `ServiceRelation`.
6. Проверить страницу `/services/{slug}` на сайте.

## Безопасный порядок публикации новости или статьи

1. Создать запись в `NewsPost` или `Article`.
2. Заполнить:
   - `title`
   - `slug`
   - `excerpt`
   - `content`
   - `publishedAt`
   - `status = PUBLISHED`
3. Заполнить SEO-поля, если они есть.
4. Проверить `/news/{slug}` или `/blog/{slug}`.
