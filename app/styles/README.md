# Структура CSS

Новые стили сгруппированы по назначению и подключаются в `app/layout.tsx` после legacy-стилей.

- `home-page.css` — первый экран, split-изображения, главные секции, статьи на главной и базовая настройка мобильного бургера.
- `seo-content.css` — типографика H1–H6, абзацы, списки, ссылки, хлебные крошки, SEO-тексты и общий CTA.
- `layout.css` — глобальные сетки шапки, футера, поиска и результатов поиска.
- `header.css` — только адаптивные правила шапки и мобильной ссылки поиска.
- `forms.css` — поля, ошибки, согласие, honeypot и состояния форм.
- `content-pages.css` — каталог услуг, страницы контактов, related-блоки, FAQ и отзывы.
- `accessibility.css` — служебные классы доступности, включая `sr-only`.

## Legacy-файлы

В корне `app/` пока остаются файлы, перенесённые из старой Webflow-вёрстки и ранних этапов адаптации:

- `globals.css`
- `ui-fixes.css`
- `page-hotfixes.css`
- `lead-page-fixes.css`
- `site-polish.css`
- `home-reference.css`
- `client-revision.css`
- `client-revision-patch.css`
- `visual-polish.css`
- `route-polish.css`
- `light-blocks-hero-fix.css`

Их нельзя массово удалять или переименовывать без визуальной регрессии. Новые правила следует добавлять в тематические файлы из `app/styles/`, постепенно перенося туда нужные селекторы из legacy-файлов.

Порядок подключения важен: тематические стили должны идти после legacy CSS, чтобы контролируемо переопределять старую вёрстку.
